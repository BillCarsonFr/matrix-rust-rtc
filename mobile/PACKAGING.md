# Mobile Packaging Documentation

This directory contains scripts and configuration for building and packaging the Matrix RTC FFI library for Android and iOS.

## Structure

```
mobile/
├── android/
│   ├── gradle/
│   │   └── wrapper/
│   │       └── gradle-wrapper.properties
│   ├── matrixrtc/
│   │   ├── src/main/
│   │   │   ├── AndroidManifest.xml
│   │   │   └── jniLibs/        (generated by build script)
│   │   ├── build.gradle
│   │   ├── proguard-rules.pro
│   │   └── consumer-rules.pro
│   ├── build.gradle
│   ├── settings.gradle
│   └── gradlew
├── ios/
│   ├── build/                  (generated by build script)
│   │   ├── MatrixRtcFFI.xcframework
│   │   └── libmatrix_rtc_ffi_sim.a
│   ├── generated/              (generated by build script)
│   │   └── matrixrtcffi.swift
│   ├── Package.swift
│   └── MatrixRtcFFI.swift
└── scripts/
    ├── build-android-aar.sh
    └── build-ios-xcframework.sh
```

## Android AAR Build

To build the Android AAR library:

```bash
./scripts/build-android-aar.sh
```

This script will:
1. Install `cargo-ndk` if needed
2. Add required Rust targets
3. Build native libraries for arm64-v8a, armeabi-v7a, and x86_64
4. Generate Kotlin bindings from UniFFI
5. Build the AAR using Gradle

**Output:** `mobile/android/matrixrtc/build/outputs/aar/matrixrtc-release.aar`

### Prerequisites

- Android NDK (set via `$ANDROID_NDK_HOME` or `$ANDROID_HOME/ndk/<version>`)
- JDK 8 or higher
- Rust with targets: `aarch64-linux-android`, `armv7-linux-androideabi`, `x86_64-linux-android`

### Usage in Android App

Add to your app's `build.gradle`:

```gradle
dependencies {
    implementation files('../path/to/matrixrtc-release.aar')
}
```

Or publish to a Maven repository and depend on it:

```gradle
dependencies {
    implementation 'org.matrix.rtc:matrixrtc:0.1.0'
}
```

## iOS XCFramework Build

To build the iOS XCFramework:

```bash
./scripts/build-ios-xcframework.sh
```

This script will:
1. Add required Rust targets for iOS
2. Build static libraries for:
   - `aarch64-apple-ios` (device)
   - `aarch64-apple-ios-sim` (simulator arm64)
   - `x86_64-apple-ios` (simulator x86)
3. Create a universal simulator library
4. Create the XCFramework
5. Generate Swift bindings from UniFFI

**Output:**
- XCFramework: `mobile/ios/build/MatrixRtcFFI.xcframework`
- Swift bindings: `mobile/ios/generated/matrixrtcffi.swift`

### Prerequisites

- Xcode (with command-line tools)
- Rust with targets: `aarch64-apple-ios`, `aarch64-apple-ios-sim`, `x86_64-apple-ios`

### Usage in iOS App

#### Option 1: SPM (Swift Package Manager)

The `Package.swift` in the iOS directory is pre-configured. After building:

```bash
# In your iOS project
git clone <your-repo> MatrixRtcFFI
# Add to your Xcode project: File > Add Packages > Select MatrixRtcFFI folder
```

#### Option 2: Manual Xcode Integration

1. Copy `mobile/ios/build/MatrixRtcFFI.xcframework` to your Xcode project
2. In Xcode: Build Phases > Link Binary With Libraries > Add MatrixRtcFFI.xcframework
3. Copy Swift bindings from `mobile/ios/generated/matrixrtcffi.swift` into your project

## Full Build Workflow

To build both Android AAR and iOS XCFramework:

```bash
# Install tools
cargo install uniffi_bindgen cargo-ndk

# Build for Android
./scripts/build-android-aar.sh

# Build for iOS
./scripts/build-ios-xcframework.sh
```

## Environment Setup

### Android NDK

Set the Android NDK location:

```bash
export ANDROID_NDK_HOME=/path/to/android-ndk
# or
export ANDROID_HOME=/path/to/android-sdk
```

The build script will auto-detect from `$ANDROID_HOME/ndk/<version>` if not explicitly set.

### Rust Targets

Ensure all required Rust targets are installed:

```bash
# iOS
rustup target add aarch64-apple-ios aarch64-apple-ios-sim x86_64-apple-ios

# Android
rustup target add aarch64-linux-android armv7-linux-androideabi x86_64-linux-android
```

## Troubleshooting

### Android Build Fails

- Check that `ANDROID_NDK_HOME` is set correctly
- Verify Rust targets: `rustup target list | grep android`
- Run `./gradlew clean` before retrying

### iOS Build Fails

- Verify Xcode command-line tools: `xcode-select --install`
- Check Rust targets: `rustup target list | grep apple`
- Ensure `uniffi-bindgen` is installed: `cargo install uniffi_bindgen`

## Publishing

### Android (Maven Central / MavenCentral)

1. Configure signing in `mobile/android/build.gradle`
2. Upload AAR to Maven repository

### iOS (CocoaPods / SPM)

1. Tag a release in git
2. Update `Package.swift` with the release version
3. Publish to CocoaPods registry or use SPM directly

## CI/CD Integration

These scripts are designed to work in CI environments. Example GitHub Actions workflow:

```yaml
name: Build Mobile Libraries

on: [push, pull_request]

jobs:
  build-android:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions-rs/toolchain@v1
      - run: ./scripts/build-android-aar.sh
      - uses: actions/upload-artifact@v3
        with:
          name: android-aar
          path: mobile/android/matrixrtc/build/outputs/aar/

  build-ios:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions-rs/toolchain@v1
      - run: ./scripts/build-ios-xcframework.sh
      - uses: actions/upload-artifact@v3
        with:
          name: ios-xcframework
          path: mobile/ios/build/MatrixRtcFFI.xcframework/
```
